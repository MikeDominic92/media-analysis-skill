## Documentation Standards

All documentation follows these principles:

1. **No Emojis:** Plain text markers ([+], [!], [*]) only
2. **Clear Structure:** Hierarchical sections with ToC
3. **Code Examples:** Practical, tested examples
4. **Cross-References:** Links to related documentation
5. **Audience-Specific:** Tailored to reader's role
